# SmartCourier Test Cases

## Overview

This project implements **62 endpoints** across 4 business services:

- Auth Service: 14 endpoints
- Delivery Service: 22 endpoints
- Tracking Service: 11 endpoints
- Admin Service: 15 endpoints

Use gateway base URL for integrated testing:

- `http://localhost:8080/gateway`

Use direct service URLs for service-specific debugging:

- Auth: `http://localhost:8081`
- Delivery: `http://localhost:8082`
- Tracking: `http://localhost:8083`
- Admin: `http://localhost:8084`

## Common Test Preconditions

- Eureka Server is running
- Config Server is running
- Auth, Delivery, Tracking, Admin, and API Gateway are up
- MySQL is running
- RabbitMQ is running
- Zipkin is running
- JWT token is generated by login for secured gateway requests

## Auth Service Test Cases

### 1. `POST /auth/signup`
- Valid case: create user with username and password, expect `200`
- Validation case: blank username or password, expect `400`
- Duplicate case: existing username, expect `401`
- Role default case: omit role, expect user role becomes `CUSTOMER`

### 2. `POST /auth/login`
- Valid case: correct username and password, expect JWT token in response
- Invalid password case: expect `401`
- Unknown user case: expect `401`

### 3. `GET /auth/users`
- Valid admin token case: expect all users list
- Non-admin token case: expect `401`
- Missing token case: expect `401`

### 4. `GET /auth/users/{id}`
- Valid id case: expect user object
- Invalid id case: expect `404`

### 5. `GET /auth/users/username/{username}`
- Valid username case: expect user object
- Invalid username case: expect `404`

### 6. `GET /auth/users/role/{role}`
- Valid role case: expect filtered users
- Unknown role case: expect empty list

### 7. `GET /auth/users/count`
- Valid case: expect total user count

### 8. `DELETE /auth/users/{id}`
- Valid id case: expect success message
- Invalid id case: expect `404`

### 9. `PUT /auth/users/{id}`
- Valid update case: update username/password/role, expect updated user
- Invalid id case: expect `404`
- Blank values case: unchanged fields should remain unchanged

### 10. `PUT /auth/users/{id}/role`
- Valid role update case: expect updated role in uppercase
- Invalid id case: expect `404`
- Blank role case: expect validation failure or handled error

### 11. `GET /auth/validate`
- Valid token case: expect `username`, `role`, and `valid=true`
- Missing token case: expect `401`
- Invalid token case: expect `401`

### 12. `GET /auth/role`
- Valid token case: expect role string
- Missing token case: expect `401`

### 13. `GET /auth/me`
- Valid token case: expect current username and role
- Missing token case: expect `401`

### 14. `POST /auth/logout`
- Valid case: expect success message

## Gateway Test Cases

These are integration-focused because gateway routes other services.

### 1. `/gateway/auth/**`
- Login/signup should work without token
- Protected auth endpoints should require token where controller expects it

### 2. `/gateway/deliveries/**`
- Valid customer/admin token should work
- Missing token should return `401`
- Unauthorized role should return `403`

### 3. `/gateway/tracking/**`
- Valid token flow should work if route is enabled through gateway usage

### 4. `/gateway/admin/**`
- Admin token should work
- Customer token should return `403`

## Cross-Service Integration Test Cases

### 1. Auth + Gateway
- Signup user
- Login
- Use returned JWT on gateway delivery APIs

### 2. Delivery + RabbitMQ + Tracking
- Create delivery
- Update delivery status to `SHIPPED`
- Confirm queue activity in RabbitMQ
- Confirm tracking record is auto-created

### 3. Admin + Feign + Delivery
- Create some deliveries
- Call admin dashboard
- Confirm counts reflect delivery-service data

### 4. Zipkin Observability
- Trigger request through gateway
- Confirm trace appears in Zipkin for `api-gateway` and downstream service

### 5. Eureka Discovery
- Start all services
- Confirm all services show `UP` in Eureka

### 6. Config Server
- Open `http://localhost:8888/auth-service/default`
- Confirm config is served successfully
